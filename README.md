# MarkItDown — конвертер файлов в Markdown для iPhone, Windows и Mac

**MarkItDown** превращает любой файл — PDF, Word (docx), PowerPoint (pptx), Excel (xlsx/xls),
письмо из Outlook (.msg) — в чистый, читаемый Markdown без мусора форматирования. Работает как
приложение на iPhone (устанавливается прямо из Safari, без App Store), как портативная
программа для Windows и как приложение для Mac — с одним общим архивом конвертированных файлов
на всех устройствах.

## Зачем это нужно

- **Получили PDF, Word- или PowerPoint-файл и хотите вставить текст в заметки, ChatGPT/Claude
  или свою markdown-базу** (Obsidian, Notion и т.п.) — без кривого форматирования, которое
  обычно остаётся при copy-paste. Просто конвертируете файл — получаете чистый текст.
- **Работает без интернета и без чужого облака.** Конвертация происходит на вашем собственном
  сервере (или прямо на компьютере в офлайн-режиме) — файлы не уходят в сторонний SaaS, платить
  за конвертацию не нужно.
- **Один архив на все устройства.** Конвертировали файл на телефоне — он же доступен на
  компьютере, и наоборот.

## Платформы

| Платформа | Папка | Как получить |
|---|---|---|

## Приложение для iPhone и iPad — без App Store

Docker-сервер (см. ниже) отдаёт progressive web app (PWA): полноценное мобильное приложение для
конвертации файлов в Markdown на iOS, которое устанавливается на экран «Домой» прямо из Safari —
без App Store, без Apple Developer аккаунта, без ревью приложений.

Что это даёт на практике:

- **Экран как у обычного App Store приложения.** После установки открывается в полноэкранном
  режиме — без адресной строки Safari, со своей иконкой на «Домой» и цветом статус-бара под тему
  интерфейса. Отличить от нативного приложения на глаз нельзя.
- **Свой сервер, а не чужое облако.** Файлы уходят на сервер, который вы сами подняли (домашний
  NAS, VPS, любой компьютер с Docker) — никакого стороннего SaaS, никакой платы за конвертацию,
  данные не покидают вашу инфраструктуру.
- **Тот же движок конвертации, что и на компьютере.** [MarkItDown](https://github.com/microsoft/markitdown)
  превращает docx, pdf, pptx, xlsx, xls, outlook (.msg) и другие форматы в чистый читаемый
  markdown прямо на телефоне — получили файл, поделились в приложение, получили markdown.
- **Архив с поиском, общий на все устройства.** Все конвертации сохраняются на сервере и видны
  с любого устройства, которое к нему подключается — iPhone, iPad, Mac, Windows-компьютер.
- **Работает даже при плохой связи.** Интерфейс кешируется на устройстве, поэтому приложение
  открывается мгновенно — сама конвертация файла, конечно, всё равно требует соединения с
  сервером.
- **Интерфейс на русском и английском** — определяется автоматически по языку системы
  устройства, переключать вручную не нужно.

Единственная разовая настройка: перед первой установкой нужно один раз одобрить HTTPS-сертификат
своего сервера (иначе Safari будет показывать предупреждение о недоверенном соединении) —
подробная инструкция в [`nas-server/README.md`](nas-server/README.md).

## Как это работает

- **Docker-сервер — источник истины.** Это сервис на Docker с HTTPS, архив хранится прямо на
  сервере. Тот же сервис отдаёт PWA-приложение для iPhone (см. выше).
- **Windows/macOS-приложения** при старте проверяют, доступен ли сервер:
  - доступен → открывают окно прямо на нём, архив общий со всеми устройствами;
  - недоступен → поднимают встроенный локальный сервер и работают офлайн, архив кладут рядом
    с приложением.
- Оболочка десктоп-приложений — [pywebview](https://pywebview.flowrl.com/)
  (Windows: WebView2, macOS: WKWebView), упаковка в один файл — PyInstaller.

## Структура репозитория

- **`nas-server/`** — Docker-сервис (`Dockerfile`, `docker-compose.yaml`, `app.py`, `static/` —
  тот же HTML/CSS/JS, что видно на iPhone как PWA). См. [`nas-server/README.md`](nas-server/README.md).
- **`main.py`** — точка входа десктоп-приложения: проверка сервера, локальный сервер, окно pywebview.
- **`server_app.py`** — FastAPI-приложение для локального (офлайн) режима десктоп-версий.
- **`static/`** — фронтенд десктоп-версий (тот же UI + бейдж режима сервер/локально).
- **`main.spec`** — PyInstaller-спека для macOS.
- **`mac-build/AppIcon.iconset/`**, **`icon.ico`** — иконка «M↓» для всех платформ.
- **`.github/workflows/build-macos.yml`** — CI, собирает `.app` под arm64 и x86_64 при пуше в `main`.
- **`DEVELOPMENT.md`** — подробности по десктоп-версии для разработки: как итерировать без
  пересборки `.app` на каждый чих, полная история известных багов и их фиксов.

## Ограничения

- Самоподписанный HTTPS-сертификат сервера нужно один раз вручную одобрить на каждом устройстве
  (iOS: профиль + полное доверие в Настройках; Windows: импорт в `CurrentUser\Root`; macOS:
  Keychain Access → «Всегда доверять»).
- Приложения не подписаны сертификатом разработчика (ни Apple, ни Microsoft) — при первом
  запуске нужно один раз подтвердить (macOS: правый клик → «Открыть»; Windows: SmartScreen,
  если появится).
- IP сервера захардкожен в `main.py` (`NAS_URL`) — при смене адреса поменять там.
