# MarkItDown — личный тулкит для конвертации файлов в Markdown

Обвязка вокруг [MarkItDown](https://github.com/microsoft/markitdown) (любой файл — pdf, docx,
pptx, xlsx, изображения и т.д. — в чистый Markdown) для трёх платформ с общим архивом
конвертированных файлов между устройствами.

## Платформы

| Платформа | Папка | Как получить |
|---|---|---|
| Docker-сервер + PWA для iOS | [`nas-server/`](nas-server/) | `docker-compose up -d --build` на любой машине с Docker (NAS, VPS, домашний сервер) |
| Windows | корень репозитория | `MarkItDown-windows-x64.exe` из [Releases](../../releases) — portable, установка не нужна |
| macOS (Intel и Apple Silicon) | корень репозитория | `.zip` из [Releases](../../releases), архитектуру выбрать под свой чип |

Папка называется `nas-server/`, потому что изначально поднималась на домашнем NAS — но это
обычный Docker-контейнер, работает на любой машине с Docker (свой сервер, VPS, что угодно).

## Как это работает

- **Docker-сервер — источник истины.** Это FastAPI-сервис на Docker с HTTPS (самоподписанный
  сертификат), архив хранится прямо на сервере. Тот же сервис отдаёт PWA — на iPhone открывается
  как обычный сайт и ставится на экран «Домой».
- **Windows/macOS-приложения** при старте проверяют, доступен ли сервер:
  - доступен → открывают окно прямо на нём, архив общий со всеми устройствами;
  - недоступен → поднимают встроенный локальный сервер и работают офлайн, архив кладут рядом
    с приложением.
- Оболочка десктоп-приложений — [pywebview](https://pywebview.flowrl.com/)
  (Windows: WebView2, macOS: WKWebView), упаковка в один файл — PyInstaller.

## Структура репозитория

- **`nas-server/`** — Docker-сервис (`Dockerfile`, `docker-compose.yaml`, `app.py`, `static/` —
  тот же HTML/CSS/JS, что видно на iPhone как PWA). См. [`nas-server/README.md`](nas-server/README.md).
- **`main.py`** — точка входа десктоп-приложения: проверка NAS, локальный сервер, окно pywebview.
- **`server_app.py`** — FastAPI-приложение для локального (офлайн) режима десктоп-версий.
- **`static/`** — фронтенд десктоп-версий (тот же UI + бейдж режима NAS/локально).
- **`main.spec`** — PyInstaller-спека для macOS (`--onedir`, ATS-исключение в `Info.plist` —
  см. «Известные грабли» ниже).
- **`mac-build/AppIcon.iconset/`**, **`icon.ico`** — иконка «M↓» для всех платформ.
- **`.github/workflows/build-macos.yml`** — CI, собирает `.app` под arm64 и x86_64 при пуше в `main`.
- **`DEVELOPMENT.md`** — подробности по десктоп-версии: как итерировать без пересборки `.app`
  на каждый чих, полная история известных багов и их фиксов.

## Известные грабли (уже починены, оставлено как документация)

Обе платформы используют один и тот же `/api/archive/{filename}` эндпоинт на сервере, но с
разным поведением для скачивания файла — обнаружено и исправлено эмпирически:

- **WKWebView (macOS) и WebView2 (Windows) решают «показать или скачать» по тому, могут ли
  отрисовать MIME-тип инлайн — и игнорируют `Content-Disposition`.** Из-за этого при заходе
  через десктоп-приложение файл открывался прямо в окне вместо сохранения на диск.
- **iOS Safari, наоборот, при `application/octet-stream` начинает сниффать реальные байты
  файла**, чтобы угадать тип для превью — и иногда принимает markdown за HTML.

Решение — различать клиента на лету через `window.pywebview` (этот объект pywebview
автоматически инжектит в каждую страницу; в настоящем браузере его никогда нет):
pywebview-клиенты запрашивают `/api/archive/{filename}?raw=1` и получают
`application/octet-stream`, обычные браузеры — `text/markdown` без параметра. Код — в
[`nas-server/app.py`](nas-server/app.py) и [`nas-server/static/app.js`](nas-server/static/app.js).

Ещё один нюанс — macOS-сборка **обязательно `--onedir`, не `--onefile`**: PyInstaller
распаковывает `--onefile`-бандл заново при каждом запуске, и на медленной машине это не
укладывается в таймаут поднятия локального сервера. Подробности — в `DEVELOPMENT.md`.

## Ограничения

- Самоподписанный HTTPS-сертификат сервера нужно один раз вручную одобрить на каждом устройстве
  (iOS: профиль + полное доверие в Настройках; Windows: импорт в `CurrentUser\Root`; macOS:
  Keychain Access → «Всегда доверять»).
- Приложения не подписаны сертификатом разработчика (ни Apple, ни Microsoft) — при первом
  запуске нужно один раз подтвердить (macOS: правый клик → «Открыть»; Windows: SmartScreen,
  если появится).
- IP сервера захардкожен в `main.py` (`NAS_URL`) — при смене адреса поменять там.
